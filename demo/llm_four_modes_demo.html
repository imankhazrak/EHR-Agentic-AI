Four LLM prediction modes

MIMIC-III task: from the current visit narrative only, predict whether Disorders of Lipid Metabolism (CCS-53) will appear on the next admission. The true label never includes next-visit text for the model. None of these modes fine-tunes weights.

Overview

Same inference stack, different prompts

Mode is selected in src/llm/predictor.py → Jinja template under prompts/. One completion per test row (plus CoAgent’s extra calls on a train calibration slice first).

zero_shot zero_shot_plus few_shot coagent
Shared

What every mode shares

  • Input column: narrative_current (bulleted diagnoses / meds / procedures)
  • System message: medical expert role (same idea across templates)
  • Target definition + “prediction clarification” (e.g. don’t say Yes from statins/CHF alone)
  • Parsed output: first line Prediction: Yes or No → src/llm/output_parser.py

Few-shot

Template: predictor_few_shot.txt

Same rich framing as zero-shot+–style instructions (including prevalence), plus [Demonstration Cases] filled by format_exemplar_block() in src/data/exemplar_selector.py.

Default: 3 positive + 3 negative from train 1× LLM / test row

EHR-CoAgent

Template: predictor_coagent_base.txt

Few-shot-style prompt with an extra section [Refined Criteria] ← {{ critic_feedback }} (consolidated text from a critic pass on calibration errors).

Same 6 demonstration cases as few-shot 1× LLM / test row + many calls on train calibration before test
EHR-CoAgent

Offline pipeline (src/llm/coagent.py)

1. Sample calibration set from train (default n=200)
2. Run few-shot predictor on calibration → merge labels
3. Keep mispredictions; sample up to n_wrong (e.g. 30)
4. Critic LLM: critic_batch_reflection.txt — batches × rounds
5. Consolidate (LLM or heuristic) → single feedback string
6. Test: run_predictions(…, mode="coagent", critic_feedback=…)
EHR-CoAgent

Extra prompt sections

[Demonstration Cases]
{{ demonstration_cases }}

[Refined Criteria]
{{ critic_feedback }}

… apply [Refined Criteria] to avoid common errors …

If calibration has no errors, pipeline falls back to plain few-shot on test.

Recap

No fine-tuning in any mode

Differences are prompt text and, for CoAgent, additional LLM calls over a labeled calibration slice to produce critic feedback. Eval still uses held-out test.csv labels (label_lipid_disorder).

Reference — per-mode detail

Zero-shot

  • Goal: Smallest prompt — rules + current visit only.
  • Variables: narrative (via Jinja).
  • Script: python -m src.scripts.run_zero_shot

Zero-shot+

  • Goal: Richer reasoning scaffold + explicit base rate, still no in-context labels from train.
  • Variables: narrative only.
  • Script: python -m src.scripts.run_zero_shot_plus

Few-shot

  • Goal: In-context learning from fixed labeled narratives (random balanced sample).
  • Variables: narrative, demonstration_cases.
  • Selection: select_exemplars() — default n_positive=3, n_negative=3, strategy=random_balanced.
  • Script: python -m src.scripts.run_few_shot

EHR-CoAgent

  • Goal: Same few-shot context plus instructions distilled from errors on a train calibration subset.
  • Variables: narrative, demonstration_cases, critic_feedback.
  • Defaults: calibration_size=200, n_wrong_samples=30, critic_batch_size=10, n_critic_rounds=3, consolidation_method=llm.
  • Prompts: critic_batch_reflection.txt, feedback_consolidation.txt.
  • Script: python -m src.scripts.run_coagent
